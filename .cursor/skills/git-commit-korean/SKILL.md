# 한국어 Git 커밋 메시지

## 커밋 메시지 형식

다음 형식을 사용한다:

```
[타입] 변경 내용 요약

- 상세 변경 사항 1
- 상세 변경 사항 2
```

- **첫 줄**: `[타입]` + 공백 + 요약 (한 줄, 명확하게)
- **빈 줄** 한 줄
- **본문**: `- `로 시작하는 상세 변경 사항 (필요 시 여러 줄)

## 커밋 타입

| 타입 | 용도 |
|------|------|
| `[기능]` | 새로운 기능 추가 |
| `[수정]` | 버그 수정 |
| `[개선]` | 기존 기능 개선 |
| `[리팩토링]` | 코드 리팩토링 |
| `[문서]` | 문서 수정 |
| `[스타일]` | 코드 포맷팅, 세미콜론 등 스타일만 변경 |
| `[테스트]` | 테스트 코드 추가/수정 |
| `[설정]` | 설정 파일 변경 |
| `[의존성]` | 패키지 추가/업데이트/삭제 |

## 예시

**예시 1**  
변경: 사용자 인증(JWT) 추가

```
[기능] JWT 기반 사용자 인증 추가

- 로그인 엔드포인트 추가
- 토큰 검증 미들웨어 구현
```

**예시 2**  
변경: 날짜 표시 버그 수정

```
[수정] 타임존 변환 시 날짜 표시 오류 수정

- 보고서 생성 시 UTC 타임스탬프로 통일
```

**예시 3**  
변경: PDF 처리 로직만 정리

```
[리팩토링] PDF 추출 로직 분리 및 함수 정리

- extract_text() 함수 분리
- 에러 처리 공통화
```

**예시 4**  
변경: README 업데이트

```
[문서] README 설치 방법 및 하드웨어 스펙 보완

- 플랫폼별 빌드 방법 추가
- IMU 센서 설정 절차 정리
```

## 작업 순서

1. `git status` / `git diff` 등으로 변경 내용 파악
2. 변경 성격에 맞는 **타입** 선택
3. 요약 줄 작성 (한국어, 간결하게)
4. 필요하면 본문에 `- ` 항목으로 상세 기술
